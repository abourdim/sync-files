📋 Overview — What is SyncFiles?

SyncFiles keeps your project files synchronized across multiple destinations:

DestinationWhat it doesYou need
☁️ Google Drive Syncs files to a Google Drive folder Google account + API credentials
📦 Git Auto-commits & pushes changes A Git repository (local or remote)
🖥️ SSH / SFTP Syncs files to a remote server SSH access (password or key)
📁 Local Copies files to another folder A target folder path
ℹ️ You can use multiple destinations at the same time. For example: Google Drive + Git + Local backup.

🚀 First Launch

1 Open Terminal

Navigate to the SyncFiles folder:

cd /path/to/sync-files

2 Run the launcher

bash sync.sh

This opens the main menu. On first run, choose option 1 — Setup Wizard.

💡 Works on both macOS and Ubuntu/Linux — no changes needed.

3 Setup Wizard walks you through

  1. Install Python dependencies
  2. Set your source folder (the project you want to sync)
  3. Choose a destination (Google Drive, Git, SSH, or Local)
  4. Enable Git integration (if a repo is detected)
  5. Create the encrypted credential store (sets a master password)
ℹ️ You can also skip the wizard and configure everything manually — see the Config section below.

4 Launch the Web Dashboard

From the menu, choose option 2 — Web Dashboard, or run directly:

bash sync.sh --web

Your browser opens to http://127.0.0.1:8765 — the SyncFiles dashboard.

☁️ Google Drive Setup

This is the most involved setup because Google requires API credentials. Follow each step carefully.

1 Create a Google Cloud Project

  1. Go to console.cloud.google.com
  2. Sign in with your Google account
  3. Click the project dropdown (top-left) → New Project
  4. Name it anything (e.g. my-syncfiles) → click Create
  5. Wait a few seconds, then select your new project from the dropdown
💡 This is free. Google Cloud projects don't cost anything for personal API use.

2 Enable the Google Drive API

  1. In your project, go to the side menu: APIs & Services → Library
  2. Search for "Google Drive API"
  3. Click on it → click the Enable button

3 Configure the OAuth Consent Screen

  1. Go to APIs & Services → OAuth consent screen
  2. Choose External → click Create
  3. Fill in the required fields:
    • App name: SyncFiles (or anything you want)
    • User support email: your email
    • Developer contact email: your email
  4. Click Save and Continue through the remaining pages (Scopes, Test users, Summary)

4 Add Yourself as a Test User

  1. Go to OAuth consent screen → Test users (or Audience)
  2. Click Add users
  3. Enter your Gmail address (the one you'll authorize with)
  4. Click Save
⚠️ If you skip this step, Google will show "Access Blocked" when you try to authorize. Your app is in test mode, so only listed test users can use it.

5 Create OAuth Client Credentials

  1. Go to APIs & Services → Credentials
  2. Click + Create Credentials → OAuth client ID
  3. Application type: Desktop app
  4. Name: anything (e.g. syncfiles-desktop)
  5. Click Create

6 Download the JSON Key

  1. After creating the client, you'll see it listed under OAuth 2.0 Client IDs
  2. Click the download icon (⬇️) on the right side of your client row
  3. Or click your client name → click Download JSON at the top
  4. Rename the downloaded file to client_secret.json
  5. Move it to the SyncFiles root folder:
mv ~/Downloads/client_secret_*.json /path/to/sync-files/client_secret.json
⚠️ Never commit client_secret.json to Git! It's already in .gitignore for safety.

7 Authorize the App

Run the authorization script:

# Activate the virtual environment first
source .venv/bin/activate

# Run authorization
python -c "
from server.credentials import CredentialStore
from server.gdrive import GDriveSync
cs = CredentialStore('.credentials')
cs.unlock('')  # use your master password instead of ''
gd = GDriveSync(cs, 'SyncFiles')
gd.authorize()
print(gd.test_connection())
"
  1. A browser window opens → Sign in with the same Google account you added as test user
  2. Click Continue when asked to grant permissions
  3. You'll see "The authentication flow has completed. You may close this window."
  4. Back in the terminal, you should see: {'ok': True, 'user': 'you@gmail.com'}
✅ Done! The token is now securely stored in the encrypted credential store. You won't need to authorize again unless the token expires.

8 Add Google Drive to config.yaml

Open config.yaml and add a Google Drive destination:

sync:
  source: '/path/to/your/project'
  destinations:
    - type: gdrive
      folder_name: SyncFiles   # folder name in your Google Drive
ℹ️ The folder_name is the Google Drive folder where files will be uploaded. It's created automatically if it doesn't exist.

9 Test It

Start the dashboard and check the Google Drive indicator:

bash sync.sh --web

The Google Drive pill should show a ● green dot. Click Sync Now to test!

🔧 Troubleshooting Google Drive
ProblemSolution
"Access Blocked" error Go back to Step 4 and add your email as a test user
"client_secret.json not found" Make sure the file is in the SyncFiles root folder (not a subfolder)
Grey dot on dashboard Restart the server: bash sync.sh --web
"Credential store locked" The store needs to be unlocked with your master password. Re-run the wizard.
Token expired Re-run Step 7 (authorize again)

📦 Git Setup

1 Make Sure Your Project is a Git Repo

If your source folder is already a Git repository, SyncFiles detects it automatically. If not:

cd /path/to/your/project
git init
git remote add origin https://github.com/you/your-repo.git

2 Enable Git in config.yaml

git:
  enabled: true        # turn on Git integration
  auto_commit: true   # auto-commit after each sync
  auto_push: false    # auto-push after commit (set true if you want)
  branch: main        # branch to use
  commit_template: 'sync: {timestamp} — {files_changed} files'

3 Understanding the Options

OptionWhat it doesRecommended
enabled Activates the Git panel on the dashboard true
auto_commit Automatically commits changes after a sync runs true for backups
auto_push Automatically pushes commits to the remote after committing false (push manually when ready)
branch Which branch to commit on main
commit_template Customize the auto-commit message. Use {timestamp} and {files_changed} as placeholders Default is fine

4 Using Git from the Dashboard

Once enabled, the dashboard shows a GIT panel where you can:

  • View current branch & status
  • See recent commit history
  • View file diffs
  • Manually commit, push, or pull
💡 Even with auto-commit off, you can always click Commit or Push manually from the dashboard.

🖥️ SSH / SFTP Setup

1 Prerequisites

  • A remote server you can SSH into
  • SSH access via password or SSH key (key recommended)
  • Know the server's hostname, port, and username

2 Generate an SSH Key (if you don't have one)

# Check if you already have a key
ls ~/.ssh/id_rsa.pub

# If not, generate one
ssh-keygen -t rsa -b 4096 -C "your@email.com"

# Copy it to the remote server
ssh-copy-id user@your-server.com

3 Add SSH Destination to config.yaml

sync:
  destinations:
    - type: ssh
      name: my-server         # a label for this destination
      host: your-server.com   # server hostname or IP
      port: 22                 # SSH port (usually 22)
      username: user           # your SSH username
      key_path: ~/.ssh/id_rsa  # path to your private key
      remote_path: /var/www/project  # where to sync on the server

4 Store SSH Password (if using password auth)

If you use a password instead of a key, store it securely in the credential store:

bash sync.sh --creds

Choose to add SSH credentials. They'll be encrypted with your master password.

💡 SSH keys are more secure and don't require storing passwords. Use keys whenever possible.

📁 Local Backup Setup

1 Add a Local Destination

The simplest destination — just copies files to another folder on your computer or an external drive.

sync:
  source: '/Users/you/Projects/my-app'
  destinations:
    - type: local
      name: backup
      path: '/Volumes/USB-Drive/backups/my-app'  # target folder
ℹ️ The target folder is created automatically if it doesn't exist.

2 That's it!

No credentials, no API keys. Just set the path and sync. 🎉

⚙️ Configuration Reference

config.yaml — Full Reference

# ── Source: the folder you want to sync ──
sync:
  source: '/path/to/your/project'

  # ── Destinations (you can have multiple) ──
  destinations:
    - type: gdrive
      folder_name: SyncFiles
    - type: local
      path: '/backup/my-project'

  # ── Sync behavior ──
  watch: true          # watch files for real-time changes
  interval: 5        # seconds between auto-syncs
  chunk_size: 4194304 # 4MB — for delta sync (advanced)
  debounce: 1.0      # wait 1s after last change before syncing

# ── Server settings ──
server:
  host: 127.0.0.1      # localhost only (safe)
  port: 8765            # dashboard port
  bind_network: false  # true = accessible from other devices
  auto_open_browser: true
  session_timeout: 1800  # 30 minutes

# ── Git (see Git section above) ──
git:
  enabled: false
  auto_commit: false
  auto_push: false
  branch: main
  commit_template: 'sync: {timestamp} — {files_changed} files'

# ── Logging ──
log:
  file: sync.log
  max_size: 10485760  # 10MB
  level: info          # info or debug

.syncignore — Exclude Files from Sync

Works like .gitignore. Create a .syncignore file in your project root:

# Folders
.venv/
node_modules/
__pycache__/

# Files
*.pyc
*.log
.DS_Store
.env

# Patterns
*.tmp
*.bak

🔐 Credential Store

How It Works

SyncFiles stores sensitive data (Google tokens, SSH passwords) in an encrypted vault:

  • Encryption: AES-256 (Fernet) — military-grade
  • Key derivation: PBKDF2 with 100,000 iterations
  • Location: .credentials/store.enc
  • Protected by: a master password you set during setup
ℹ️ Your passwords and tokens are never stored in plain text. Even if someone copies the file, they can't read it without your master password.

Managing Credentials

# Open the credential manager
bash sync.sh --creds

From here you can:

  • View stored services (Google Drive, SSH, etc.)
  • Add new credentials
  • Remove old credentials
  • Change the master password

❓ FAQ

Can I use multiple destinations at once?

Yes! Add multiple entries under destinations: in config.yaml. For example, sync to Google Drive AND a local backup folder simultaneously.

Is my data secure?

Yes. Credentials are AES-256 encrypted. The dashboard runs on localhost only by default — no one on your network can access it unless you enable bind_network: true.

What happens if there's a conflict?

SyncFiles detects when a file has been modified in both the source and a destination. It shows the conflict in the dashboard with a side-by-side diff viewer. You choose which version to keep. A backup of the other version is saved in .backups/.

Does it work without internet?

Yes — for local and Git (local) destinations. Google Drive and SSH obviously require a network connection, but the dashboard and local sync work fully offline.

How do I update SyncFiles? 
bash sync.sh --update

Or pull the latest from Git if you cloned the repo.

How do I reset everything? 
bash sync.sh --clean

This removes temp files, logs, and state. Your config and credentials are preserved. To fully reset, delete .credentials/ and re-run the wizard.
